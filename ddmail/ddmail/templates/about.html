About

Here you can find information regarding ddmail e-mail system/service.

What is ddmail.se

E-mail system/service for reasonable paranoid people who value privacy and security.

Privacy

We try to honner your privacy. We do not track you. We do not log or save your ip. We do not serve ads. We do not use javascript. The service is avalible as a tor onion service(before known as tor hidden service). The service is avalible as a i2p service. No personal information is needed for registration. We provide privacy friendly payment options.

Security

We try to use the following design patterns/principle:
- less code, less attacksurface
- leased privileged
- default deny
- assume breach
The server configuration and application code is all open source and can be reviewed and used by anyone at https://github.com/drzobin. We see all input as evil. We use hard to crack password hashing with argon2. We autoupdate the server software every day. We use mod_security as web application firewall (WAF). We use namespaces and seccomp to sandbox processes. We do not provide any webmail interface.

Security regarding SMTP and DNS

We are using DNSSEC, SPF, DKIM, DMARC, CAA and MTA-STS to make the service more resilient to State-Sponsored threats.

Encryption "at rest"

E-mail is encrypted "at rest" using elliptic curve secp521r1 password protected private keys. The private key password is the email account password base64 encoded. This encryption/decryption is done seamlessly to the user on the server. Every e-mail account(Maildir) has different private/public keys.

Encryption "in transit"

E-mail is encrypted "in transit" as offen as possible using TLS encryption. IMAP and HTTP is always encryptet using TLS 1.2 or TLS 1.3 with strong cipher suites, A+ rating on ssllab. SMTP traffic is encrypted using TLS when the counter part SMTP server supports it.

Features

Unlimited number of e-mails accounts per account. Unlimited number of aliases per account. Unlimited number of own domains per account. Avalible as Tor Onion Service. Avalible as i2p service. Protocol supported for getting emails is imaps and protocol supported for sendning email is smtp.
